Lecture 6 & 7: Inheritance

المحاضرتان 6 و 7: الوراثة (Inheritance)

الشرح التفصيلي لجميع شرائح المحاضرة (Slide-by-Slide) مدعوم بشرح الأكواد سطراً بسطر (Line-by-Line Code Explanation)

Slide 3: مفهوم الوراثة (What is Inheritance?)

مبدأ الـ Inheritance (الوراثة) هو واحد من أهم عواميد الـ OOP. الفكرة ببساطة إنك بدل ما تبني كلاس جديد من الصفر، إنت بتورث كلاس تاني موجود بالفعل وتاخد كل الخصائص (Attributes) والدوال (Methods) بتاعته على الجاهز، وتزود عليهم الحاجات الجديدة بتاعتك.

عشان كده عندنا مصطلحين أساسيين:

  • Base Class (الكلاس الأب/الأساسي): الكلاس القديم اللي هنورث منه (زي كلاس Employee).
  • Derived Class (الكلاس الابن/المشتق): الكلاس الجديد اللي بياخد خصائص الأب ويزود عليها (زي كلاس Manager).

Slide 4: علاقة الـ Is-A (The "Is-A" Relationship)

إمتى نستخدم الوراثة؟ بنستخدمها لما تكون العلاقة بين التو كلاسس علاقة "هو عبارة عن" أو (Is-A).

أمثلة:

  • السيارة (Car) Is-A مركبة (Vehicle). إذن Car تورث من Vehicle.
  • المدير (Manager) Is-A موظف (Employee). إذن Manager يورث من Employee.
  • الكلب (Dog) Is-A حيوان (Animal). إذن Dog يورث من Animal.
💡 الفائدة الأساسية (Code Reusability): الوراثة بتوفر علينا كتابة الكود المكرر. بدل ما نكتب دالة (يمشي) لكل الحيوانات، بنكتبها مرة واحدة في الأب (Animal)، وكل الأبناء بيورثوها أوتوماتيك!

Slide 6: طريقة كتابة الوراثة (Inheritance Syntax)

عشان نخلي كلاس يورث من كلاس تاني في الـ C++، بنستخدم النقطتين فوق بعض : (Colon).

// 1. Base Class (Parent) class Shape { public: void setWidth(int w) { width = w; } void setHeight(int h) { height = h; } protected: int width; int height; }; // 2. Derived Class (Child) inheriting from Shape class Rectangle : public Shape { public: int getArea() { return (width * height); // It can use width & height because it inherited them! } };
  • الكلاس Rectangle اتكتب جنبه : public Shape. ده معناه إنه بيورث كلاس Shape وراثة عامة.
  • الابن Rectangle قدر يستخدم المتغيرات width و height اللي موجودين في الأب مع إنه هو نفسه معندهوش المتغيرات دي!

Slide 10: الكلمة السحرية (The 'protected' Access Modifier)

إحنا عارفين الـ public (أي حد يشوفه) والـ private (محدش يشوفه غير الكلاس نفسه).

الوراثة جابت معاها نوع تالت اسمه protected (محمي).

الـ protected بيشتغل إزاي؟

  • بالنسبة للـ main() والناس اللي بره: هو زيه زي الـ private بالظبط (ممنوع اللمس).
  • بالنسبة للكلاسات الأبناء (Derived Classes): هو زيه زي الـ public يقدروا يستخدموه ويعدلوا فيه براحتهم!

يعني ده مستوى حماية "عائلي"، الأب بيسمح لولاده بس يوصلوا لحاجته، لكن الغريب لأ.

Quiz 1: Inheritance Basics & Protected

1. What is the primary advantage of using Inheritance in OOP?

Correct Answer: B. Inheritance allows derived classes to reuse the code (attributes and methods) already written in the base class.

2. Which access specifier should be used in a Base class if you want the data to be hidden from the `main()` function, but fully accessible to its Derived (child) classes?

Correct Answer: C. `protected` acts like private for the outside world, but acts like public for inherited children.

Slide 13: ترتيب البناء والهدم في الوراثة (Constructors & Destructors Order)

لما تعمل Object من الكلاس الابن (Derived)، هل الابن بيتبني الأول ولا الأب؟ (سؤال امتحان مؤكد).

المنطق بيقول: مفيش ابن بيتولد قبل أبوه!

  • أثناء الإنشاء (Constructors): الكومبايلر بيبدأ بتشغيل Constructor الكلاس الأب (Base) الأول، وبعدين يخلص وينزل يشغل Constructor الكلاس الابن (Derived).
  • أثناء الهدم (Destructors): بنعكس القاعدة (LIFO)! الابن هو اللي بيموت ويتهد الأول، وبعدين الأب يتهد بعده. (الـ Derived Destructor بيشتغل قبل الـ Base Destructor).

Slide 16: تمرير المتغيرات لأباء (Passing Arguments to Base Constructor)

بما إن الأب بيتبني الأول، طب لو الـ Constructor بتاع الأب ده بياخد Parameters (مش Default)؟ إزاي الابن يبعتله الـ Parameters دي وهو بيتبني؟

بنستخدم طريقة اسمها Initialization List (قائمة التهيئة). بنحط نقطتين : بعد Constructor الابن، وننادي على Constructor الأب ونديله قيمه.

class Parent { public: Parent(int x) { cout << "Parent built with " << x << endl; } }; class Child : public Parent { public: // Child Constructor passing 'p' to Parent Constructor BEFORE its own body Child(int p, int c) : Parent(p) { cout << "Child built with " << c << endl; } }; int main() { Child obj(10, 20); // 10 goes to Parent, 20 stays in Child }

Slide 19: تعدي الدوال (Function Overriding)

الـ Overriding (مش الـ Overloading!) بيحصل في الوراثة بس.

يعني إيه؟ يعني الأب عنده دالة اسمها print() بتطبع حاجة معينة، بس الابن مش عاجبه كلام أبوه وعايز يغيره! فالابن بيعمل دالة بنفس الاسم ونفس الـ Parameters بالظبط print() جوا الكلاس بتاعه.

لما تعمل Object من الابن وتنادي print()، الكومبايلر بيتجاهل دالة الأب وبيشغل دالة الابن (الابن بيطغى على الأب).

Slide 20: الوراثة المتعددة (Multiple Inheritance)

في الـ C++، الابن يقدر يورث من أكتر من أب في نفس الوقت! (دي ميزة مش موجودة في لغات تانية زي Java).

يعني ممكن نعمل كلاس SmartPhone يورث من كلاس Camera وكلاس Phone في نفس الوقت!

// Base 1 class Camera { ... }; // Base 2 class Phone { ... }; // Derived Class inheriting from BOTH! class SmartPhone : public Camera, public Phone { // Has features of both a camera and a phone };

Slide 25: مشكلة الغموض (Ambiguity in Multiple Inheritance)

الوراثة المتعددة بتعمل مشكلة خطيرة اسمها Ambiguity (الغموض).

تخيل كلاس الابن (SmartPhone) وارث من (Camera) ومن (Phone)، والاتنين الأبهات عندهم دالة بنفس الاسم اسمها turnOn()! لما الابن يقول s.turnOn()، الكومبايلر هيتلخبط ومش هيبقى عارف يشغل الدالة بتاعة الكاميرا ولا بتاعة التليفون!

الحل (Resolution): إننا نحدد اسم الكلاس الأب اللي نقصده باستخدام معامل الـ Scope Resolution ::.

SmartPhone s; // s.turnOn(); // ❌ ERROR: Ambiguous! s.Camera::turnOn(); // ✅ CORRECT: Turn on the camera s.Phone::turnOn(); // ✅ CORRECT: Turn on the phone

Final Quiz: Inheritance Mechanics

1. When creating an object of a Derived class, what is the exact order of execution for Constructors and Destructors?

Correct Answer: A. Base must be constructed first to support the Derived. Destruction happens in reverse order (LIFO).

2. In Multiple Inheritance, if two base classes have a function with the exact same name, and the derived class calls it, an 'Ambiguity Error' occurs. How do you resolve this in C++?

Correct Answer: D. You must explicitly tell the compiler which parent's function you intend to call using `::`.

الملخص الشامل لأهم التريكات (Summary & Pro Tips)

  • Inheritance = وراثة: كلاس ابن (Derived) بياخد كل خصائص ودوال كلاس أب (Base) ويزود عليها. بنستخدمها لما العلاقة تكون "Is-A".
  • Syntax: class Child : public Parent { }; ← النقطتين (:) هي اللي بتعمل الوراثة.
  • protected: مستوى حماية "عائلي" ← مخفي عن الـ main (زي private) بس متاح للأبناء (زي public). ده الأنسب لمتغيرات الأب اللي عايز الابن يوصلها.
  • ترتيب البناء: Constructor الأب بيشتغل الأول، وبعدين الابن. الهدم بالعكس (Derived Destructor أولاً ثم Base).
  • Initialization List: لو الأب عنده Parameterized Constructor، الابن بيبعتله القيم بـ Child(int p, int c) : Parent(p) { }.
  • Overriding: الابن يعمل دالة بنفس اسم ونفس Parameters بتاعة الأب ← دالة الابن بتطغى على دالة الأب.
  • Multiple Inheritance: الابن يورث من أكتر من أب: class C : public A, public B { };. لو في دالة بنفس الاسم في الأبهات بيحصل Ambiguity والحل بالـ Scope Resolution obj.A::func();.

سؤال مقالي متوقع للامتحان (Essay Question)

Question: Explain the concept of Inheritance in C++. Discuss the `protected` access modifier, the order of Constructor/Destructor calls, how to pass arguments to a Base class constructor, and the Ambiguity problem in Multiple Inheritance with its solution.

Model Answer:

Inheritance allows a Derived class to reuse code from a Base class using the "Is-A" relationship. Syntax: class Derived : public Base { };.

Protected: Acts like private for external code but like public for derived classes, providing family-level access control.

Constructor/Destructor Order: When creating a Derived object, the Base constructor runs first, then the Derived constructor. Destruction is reversed (LIFO): Derived destructor first, then Base destructor.

Passing Arguments: Use an Initialization List: Child(int p, int c) : Parent(p) { } to forward arguments to the Base constructor before the Derived body executes.

Multiple Inheritance Ambiguity: When a class inherits from two bases that share a function name, calling it causes an ambiguity error. Resolution: use the Scope Resolution Operator: obj.BaseA::func();.

🔍 Based on the following code, answer all the following questions

class Shape { protected: int width; public: Shape(int w) { width = w; } }; class Square : public Shape { public: Square(int w) : Shape(w) {} int area() { return width * width; } }; int main() { Square s(4); cout << s.area() << endl; }

a) What is the output?

16 (4 × 4 = 16)

b) Why is width declared as protected instead of private?

So that the derived class Square can access it directly. If it were private, Square would get a compilation error when trying to use width.

c) Can we write cout << s.width; in main()?

No. protected members are accessible in derived classes but NOT from external code like main().
