المحاضرة الثانية: مبادئ الـ OOP

الشرح التفصيلي لجميع شرائح المحاضرة (Slide-by-Slide) مدعوم بشرح الأكواد سطراً بسطر والتريكات الهامة

Slide 4: مبادئ الـ OOP الأربعة (OOP Principles)

في السلايد دي بنتعرف على الأعمدة الأربعة اللي بيقوم عليها عالم الـ OOP بالكامل. أي لغة برمجة عشان نقول عليها إنها بتدعم الـ OOP، لازم تدعم المبادئ دي:

  • Encapsulation (الكبسلة/التغليف): تجميع الداتا (Variables) والوظائف (Functions) اللي بتتعامل معاهم في مكان واحد (Class)، ونخبي التفاصيل الداخلية عن أي حد بره الكلاس.
  • Abstraction (التجريد): إنك تظهر للمستخدم الحاجات اللي محتاجها بس (Essential Info) وتخفي عنه التعقيدات والتفاصيل الداخلية اللي ملهاش لازمة بالنسبة له.
  • Inheritance (الوراثة): كلاس جديد بياخد (يورث) خصائص ووظائف كلاس قديم، عشان نقلل تكرار الكود (Code Reuse).
  • Polymorphism (تعدد الأشكال): إن نفس الحاجة (زي دالة معينة) يكون ليها أكتر من شكل أو أكتر من طريقة تنفيذ بناءً على الكائن اللي بيستخدمها.

كل مبدأ من دول هنمسكه بالتفصيل في السلايدز الجاية.

Slide 5: الـ Encapsulation (التغليف)

مبدأ الـ Encapsulation هو ببساطة إننا بنخفي البيانات (Information hiding) جوه الكلاس، ومحدش يقدر يوصلها من بره بشكل مباشر. بنسمح بس بالوصول ليها عن طريق دوال (Functions) معينة إحنا اللي بنحددها.

class Rectangle { int width, height; public: void set_values (int, int); int area() {return width*height;} };

شرح الكود سطراً بسطر (Line-by-Line Analysis):

  • class Rectangle { : بنعرف كلاس جديد اسمه Rectangle (مستطيل). الكلمة المفتاحية class هي اللي بنستخدمها عشان ننشئ قالب جديد.
  • int width, height; : دي خصائص المستطيل (العرض والطول). وطالما مفيش قبلهم كلمة public، فهم يعتبروا private (مخفيين) بشكل افتراضي، يعني محدش يقدر يعدلهم من بره الكلاس مباشرة. وده هو الـ Encapsulation الحقيقي!
  • public: : الكلمة دي معناها إن كل اللي جاي تحتها هيكون "عام" (متاح لأي حد بره الكلاس يستخدمه).
  • void set_values (int, int); : دالة دورها إنها تاخد قيمتين وتحطهم في الـ width والـ height. بنستخدمها عشان نقدر نغير القيم المخفية بطريقة آمنة.
  • int area() {return width*height;} : دالة بترجع لنا المساحة (الطول في العرض). بتستخدم الداتا المخفية عشان تحسب النتيجة.
  • }; : قفلة الكلاس، ولازم تنتهي بفاصلة منقوطة.

Slide 6: الـ Abstraction (التجريد)

الـ Abstraction معناه إننا بنقدم التفاصيل الأساسية للمستخدم، ونخفي عنه تفاصيل التنفيذ المعقدة. زي بالظبط لما تسوق عربية، إنت بتعرف تدوس بنزين وفرامل، بس مش محتاج تعرف الموتور بيحرق البنزين إزاي من جوه!

#include <iostream> using namespace std; class Adder { public: Adder(int i=0) { total = i; } void addNum(int number) { total += number; } int getTotal() { return total; } private: int total; };

شرح الكود سطراً بسطر (Line-by-Line Analysis):

  • class Adder { : بنعمل كلاس وظيفته إنه يجمع أرقام.
  • public: : الجزء ده هو اللي اليوزر بيشوفه ويتعامل معاه (الواجهة أو الـ Interface).
  • Adder(int i=0) { total = i; } : دي حاجة اسمها Constructor، وظيفتها إنها تدي قيمة ابتدائية للمجموع أول ما نعمل الكائن.
  • void addNum(int number) { total += number; } : دالة بتاخد رقم وتجمعه على المجموع الكلي. اليوزر بيناديها عشان يزود رقم، بس ميعرفش الجمع بيتم إزاي جوه.
  • int getTotal() { return total; } : دالة بترجع المجموع الحالي عشان اليوزر يشوفه.
  • private: : الجزء المخفي عن اليوزر! (تفاصيل التنفيذ المعقدة).
  • int total; : ده المتغير اللي بنخزن فيه المجموع. اليوزر ميقدرش يوصله مباشرة، لازم يستخدم الدالة getTotal(). ده مثال ممتاز لدمج الـ Encapsulation عشان نحقق الـ Abstraction.

Quiz 1: Encapsulation & Abstraction

1. Which OOP principle is mainly responsible for hiding the internal state (data) and requiring all interaction to be performed through an object's methods?

Correct Answer: C. Encapsulation bundles data and methods together and hides the data using access modifiers like 'private'.

2. In C++, if no access specifier (public, private, protected) is declared at the beginning of a class, the members are by default:

Correct Answer: B. Unlike a struct (which defaults to public), class members in C++ default to private.

Slide 7: الـ Inheritance (الوراثة) - الفكرة الأساسية

مبدأ الـ Inheritance أو الوراثة، بيحل لنا مشكلة تكرار الكود بشكل جذري. تخيل إن عندنا كلاس للطالب (Student) وكلاس للموظف (Employee).

هتلاقي إن الاتنين عندهم بيانات مشتركة زي: الاسم، العمر، العنوان. وعندهم دوال مشتركة زي إنهم بيتكلموا أو بياكلوا.

بدل ما نكتب الكود مرتين، الوراثة بتقولك: اعمل كلاس كبير سميه Person (شخص)، وحط فيه كل الحاجات المشتركة دي مرة واحدة.

وبعدين خلي الـ Student والـ Employee يورثوا من الـ Person، وبكده هياخدوا كل الحاجات المشتركة تلقائياً، وتزود بس الحاجات الخاصة بيهم (زي الجي بي إيه للطالب، والمرتب للموظف).

Slide 8: الـ Inheritance - المستويات (Parent & Child)

في السلايد دي بيوضح المسميات التقنية للوراثة اللي لازم تكون عارفها:

  • Parent Class (أو Base Class): ده الكلاس الأب، أو الأكثر عمومية (Most General). زي كلاس الـ Person.
  • Child Class (أو Derived Class): ده الكلاس الابن، اللي بيورث من الأب. بيكون أكثر تخصصاً (More Specific). زي كلاس الـ Student أو Employee.

الكلاس الابن بياخد (Attributes & Functions) بتاعة الكلاس الأب بالكامل، وبيقدر يضيف عليها الخاص بيه.

Slide 9: الـ Inheritance - شجرة الوراثة (Inheritance Tree)

الوراثة ممكن تكون شجرة متفرعة (Multi-level Inheritance). يعني الكلاس الابن ممكن يورثه كلاس تاني (يقى حفيد!).

  • الـ Person: هو الـ Most General (الأب الأكبر).
  • الـ Student: ورث من Person، وبقى هو نفسه أب لكلاسات تانية زي ITStudent و MathStudent.
  • الـ Employee: ورث من Person، وبقى أب لكلاس Driver (سواق) و Engineer (مهندس).

الكائن اللي بنعمله من ITStudent هيكون عنده الصفات الخاصة بـ ITStudent، زائد اللي ورثها من Student، زائد اللي ورثها من Person! شفت توفير الكود وصل لفين؟

Slide 10: الـ Inheritance - مثال الحيوانات

هنا مثال تاني كلاسيكي للتوضيح:

  • Base Class (الكلاس الأساسي): كلاس Animal (حيوان). عنده دوال مشتركة لأي حيوان زي eat() و sleep().
  • Derived Class (الكلاس المشتق/الابن): كلاس Dog (كلب). بيورث من Animal، وبنزودله دالة خاصة بيه وهي bark().

Slide 11: تطبيق الـ Inheritance بالكود (C++)

تعالوا نشوف إزاي بنكتب الوراثة فعلياً في لغة C++.

class staffmember { protected: int empolyeeid; string name, phone, email; public: void print(); double pay(); }; class empolyee : public staffmember { // employee specific data goes here };

شرح الكود سطراً بسطر (Line-by-Line Analysis):

  • class staffmember { : بنعمل كلاس أب اسمه staffmember.
  • protected: : ده مستوى وصول جديد (Access Specifier)! الـ protected معناه إن المتغيرات دي "مخفية" عن الناس اللي بره (زي الـ private بالظبط)، ولكن مسموح للكلاسات الأبناء (الـ Derived classes) إنهم يشوفوها ويستخدموها.
  • class empolyee : public staffmember : ده السطر السحري بتاع الوراثة! بنعمل كلاس جديد اسمه empolyee، وعلامة النقطتين (:) متبوعة بـ public staffmember معناها: "يا كلاس empolyee، أورث كل حاجة من كلاس staffmember".

Quiz 2: Inheritance

1. What access specifier should you use in a base class if you want the data to be hidden from the outside world, but still accessible to derived (child) classes?

Correct Answer: B. The 'protected' modifier provides a middle ground: it acts like 'private' for the outside world, but acts like 'public' for derived classes.

2. What is the correct syntax in C++ to make a class 'Dog' inherit publicly from a class 'Animal'?

Correct Answer: C. In C++, a colon (:) is used followed by the inheritance type (public/private/protected) and the base class name.

Slide 12: الـ Polymorphism (تعدد الأشكال)

الـ Polymorphism معناه حرفياً "القدرة على اتخاذ أكثر من شكل". برمجياً معناه إننا ممكن يكون عندنا نفس اسم الدالة، بس الدالة دي بتعمل أكشن مختلف (Behaviour) على حسب الكائن اللي بيناديها.

أمثلة للتوضيح:

  • عندنا كلاس أب اسمه Shape جواه دالة اسمها Draw().
  • الكلاس ده بيورثه كلاس Line، و Triangle، و Circle.
  • لما انادي دالة Draw() من كائن الدائرة، هترسم دائرة. ولما انادي نفس الدالة Draw() من كائن المثلث، هترسم مثلث! نفس الأمر، بأشكال تنفيذ مختلفة.

لو مفيش Polymorphism، كنت هضطر أعمل دوال بأسماء مختلفة: drawCircle() و drawTriangle()، وده هيخلي الكود معقد.

Slide 13: تطبيق الـ Polymorphism بالكود (C++)

في السلايد دي بنشوف مثال عملي على الـ Polymorphism باستخدام كلاسات الثدييات.

class Mammal { public: virtual void makeSound() = 0; string toString() { return "Mammal"; } }; class Cat : public Mammal { public: virtual void makeSound() { cout << "rawr" << endl; } string toString() { return "Cat"; } }; class Siamese : public Cat { public: virtual void makeSound() { cout << "meow" << endl; } string toString() { return "Siamese"; } virtual void scratchCouch() { cout << "scraaaatch" << endl; } };

شرح الكود سطراً بسطر (Line-by-Line Analysis):

  • class Mammal { : كلاس الأب (الثدييات).
  • virtual void makeSound() = 0; : الكلمة السحرية virtual بتعرف الكومبايلر إن الدالة دي "قابلة لإعادة التشكيل". والـ = 0 معناها إن دي Pure Virtual Function (يعني الكلاس الأب مش هيعملها كود، وسايب الأبناء هما اللي يكتبوها بطريقتهم).
  • class Cat : public Mammal : كلاس القطة ورث من الـ Mammal.
  • virtual void makeSound() { cout << "rawr" << endl; } : القطة هنا عملت إعادة تعريف (Override) لدالة makeSound وخليت صوتها "rawr".
  • class Siamese : public Cat : كلاس السيامي ورث من الـ Cat.
  • virtual void makeSound() { cout << "meow" << endl; } : السيامي غيرت الصوت وخليته "meow". ده هو الـ Polymorphism! نفس الدالة makeSound() بتطلع أصوات مختلفة حسب إنت بتناديها من فين.

Slide 14: فوائد الـ OOP (Benefits of using OOP)

كده نقدر نلخص أهم فوائد استخدام البرمجة كائنية التوجه (OOP):

  • Code can be reused by using inheritance: إعادة استخدام الكود بسهولة عن طريق الوراثة، فمش هنحتاج نكتب نفس الحاجة مرتين.
  • Data can be hidden from outside world by using encapsulation: إخفاء وحماية البيانات الحساسة عن طريق التغليف، فمحدش يلعب في الداتا بتاعتنا من بره.
  • Operators or functions can be overloaded by using polymorphism: إعادة استخدام نفس أسماء الدوال والأوامر لمهام مختلفة حسب الكائن عن طريق تعدد الأشكال.

Quiz 3: Polymorphism & OOP Benefits

1. What keyword in C++ is used in a base class to allow a function to be overridden in a derived class, enabling Polymorphism?

Correct Answer: B. The 'virtual' keyword tells the compiler to use dynamic linkage, allowing the correct derived class method to be called.

2. If you want to reuse existing code to build a new class with additional features, which OOP principle are you applying?

Correct Answer: C. Inheritance allows a new class (derived) to inherit the attributes and methods of an existing class (base).

الملخص الشامل لأهم التريكات (Summary & Pro Tips)

  • Encapsulation (الكبسلة): بنحققه في C++ عن طريق استخدام الـ Access Modifiers زي private عشان نخفي الداتا، و public للدوال اللي بتتعامل معاها.
  • Abstraction (التجريد): هو إنك تركز على "إيه اللي بيحصل؟" مش "إزاي بيحصل؟". بتعمله عن طريق إخفاء تفاصيل الكود خلف دوال public يقدر اليوزر يستخدمها من غير ما يفهم اللوجيك الداخلي.
  • الفرق بين private و protected: الـ private مخفي عن أي حد بره الكلاس (حتى أبناءه). لكن الـ protected مخفي عن العالم الخارجي بس متاح لأي كلاس بيورث منه (Child Class). دي نقطة بتيجي 100% في الامتحان!
  • الـ Virtual Function: عشان الكومبايلر يفهم إن دي دالة ليها أكتر من شكل (Polymorphism)، لازم تكتب قبلها كلمة virtual في الكلاس الأب.

سؤال مقالي متوقع للامتحان (Essay Question)

Question: Briefly define the four main principles of Object-Oriented Programming (OOP). Give a short real-world analogy for each.

Model Answer:

The four main principles of OOP are:

  • 1. Encapsulation: Bundling data and the methods that operate on that data into a single unit (class), while restricting direct access to some of the object's components.
    Analogy: A medical capsule; the chemical contents are hidden and protected inside the shell.
  • 2. Abstraction: Hiding complex implementation details and showing only the essential features of an object.
    Analogy: Driving a car; you know how to use the steering wheel and pedals, but you don't need to know the internal combustion engine mechanics.
  • 3. Inheritance: A mechanism where a new class (derived class) inherits properties and behaviors from an existing class (base class), promoting code reusability.
    Analogy: A child inheriting physical traits (like eye color) from their parents.
  • 4. Polymorphism: The ability of a single function or method to take on multiple forms depending on the object invoking it.
    Analogy: The command "Speak". If given to a dog, it barks. If given to a cat, it meows. The command is the same, but the execution differs.

🔍 Based on the following code, answer all the following questions

class BankAccount { private: double balance; public: void setBalance(double b) { if (b >= 0) balance = b; } double getBalance() { return balance; } void deposit(double amount) { balance = balance + amount; } }; int main() { BankAccount acc; acc.setBalance(1000); acc.deposit(500); cout << acc.getBalance() << endl; }

a) What is the output?

1500

b) What will happen if we call acc.setBalance(-500);?

Nothing will happen. The if condition (b >= 0) will be false, so balance will not be changed. This is the power of validation in setters.

c) Why is balance declared as private?

To enforce Encapsulation. It prevents external code from directly modifying the balance (e.g., setting it to a negative value). All access must go through the controlled setter/getter functions.
